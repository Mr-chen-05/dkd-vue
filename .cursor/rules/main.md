## Templates Directory 
 
The templates/ directory contains initialization templates for AI context recording: 
- project-init-template.md - Template for creating project.context.md files 
- archive-init-template.md - Template for creating context.archive.md files 
 
AI will automatically use these templates when initializing project context recording. 
Templates support variable replacement: {{PROJECT_NAME}}, {{TIMESTAMP}}, etc. 
 
## MCP Tools Directory 
 
The mcp-tools/ directory contains management scripts for AI to use: 
- mcp-cross-platform-sync.ps1 - Cross-platform MCP configuration sync 
 
AI can directly call these scripts for MCP management and orchestration. 
 
## Priority Levels for AI Understanding 
 
**AI should follow rules in this priority order:** 
- P0 (1100-1200): Core Safety - MUST follow first 
- P1 (1000-1099): Core Identity - Fundamental behavior 
- P2 (900-999): Intelligent Systems - Advanced capabilities 
- P3 (800-899): Professional Development - Domain expertise 
- P4 (700-799): Project Workflows - Operational procedures 
- P5 (600-699): Advanced Features - Enhanced functionality 
- P6 (500-599): System Optimization - Performance tuning 
- P7 (300-499): Utilities - Supporting tools
